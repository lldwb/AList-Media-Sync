# 快速验证指南：存储引擎重构与体验优化

**日期**：2026-06-20 | **功能**：006-storage-engine-refactor

## 前提条件

- Java 21 + Maven 已安装
- Node.js 18+ 已安装
- 项目已克隆到本地

## 构建与启动

```bash
# 后端构建（含前端构建）
mvn clean package -DskipTests

# 启动应用
java -jar target/alist-media-sync-*.jar

# 或开发模式
mvn spring-boot:run
```

应用启动后访问 `http://localhost:8080`

---

## 验证场景

### 场景 1：本地路径存储引擎创建与连接测试

**验证目标**：FR-002, FR-003, FR-005

1. 打开存储引擎管理页面
2. 点击"新增存储引擎"
3. 选择类型"本地路径"
4. 填写名称 `本地测试`、本地路径 `/tmp/media`（需先 `mkdir -p /tmp/media`）
5. 点击"测试连接" → 应返回成功
6. 保存 → 引擎列表中可见 `本地测试`，类型为 `LOCAL`
7. 编辑该引擎 → 类型字段应为禁用状态（灰色不可点击）

**预期结果**：本地路径引擎创建成功，连接测试通过，类型不可更改。

### 场景 2：AList 存储引擎（无用户名）

**验证目标**：FR-001, FR-004

1. 点击"新增存储引擎"
2. 选择类型"AList"
3. 填写名称、服务器地址、Token（不填写用户名）
4. 点击"测试连接" → 应返回成功
5. 保存 → 引擎列表中可见，无用户名字段

**预期结果**：AList 引擎创建无需用户名，连接测试通过。

### 场景 3：转码三步流程

**验证目标**：FR-006, FR-007

1. 准备：源存储引擎中有一个 FLV 视频文件
2. 创建转码任务：指定源引擎、源文件路径、目标引擎、目标路径、格式 MP3
3. 观察状态变化：待处理 → 下载中 → 转码中 → 上传中 → 完成
4. 转码任务列表中不应出现目录条目，仅显示文件

**预期结果**：任务按三步流程执行，状态依次变化，最终完成。

### 场景 4：转码失败重试

**验证目标**：FR-007（可恢复性）

1. 创建转码任务，等待其进入"下载失败"状态
2. 点击"重试" → 状态应回到"下载中"，重新下载
3. 创建转码任务，等待其进入"转码失败"状态
4. 点击"重试" → 状态应回到"转码中"，从已下载的源文件重新转码（无需重新下载）
5. 创建转码任务，等待其进入"上传失败"状态
6. 点击"重试" → 状态应回到"上传中"，从已转码的文件重新上传

**预期结果**：重试从失败步骤继续，跳过已完成步骤。

### 场景 5：树状目录浏览

**验证目标**：FR-013, FR-014, FR-015

1. 在存储引擎配置或同步任务配置中，点击路径输入框旁的浏览按钮
2. 树状目录组件展开，显示根目录下的子目录
3. 点击某个目录节点 → 加载该目录的子目录（仅目录，无文件）
4. 选中一个路径 → 自动填入输入框
5. 分别使用 AList 类型和 LOCAL 类型引擎测试

**预期结果**：目录树正常加载，路径回填正确，两种引擎类型均可使用。

### 场景 6：Webhook 规则路径显示与地址复制

**验证目标**：FR-010, FR-011, FR-012

1. 创建或编辑 Webhook 规则，选择"转 MP3"或"同步 + 转 MP3"动作
2. 源路径显示为"录播存储引擎（下拉选择器）→ 录播文件路径"组合
3. 目标路径显示为"目标存储引擎 → 目标文件路径"组合
4. 在 Webhook 规则列表页，点击"复制 Webhook 地址"按钮
5. 剪贴板中应包含 `{服务地址}/api/webhooks/recorder` 格式的 URL
6. 按钮短暂显示"已复制"后恢复

**预期结果**：路径显示清晰，复制功能正常。

### 场景 7：Cron 图形化配置

**验证目标**：FR-016, FR-017

1. 创建或编辑同步任务，选择 Cron 调度方式
2. 查看 Cron 配置区域：显示分钟、小时、日、月、周五个选择器
3. 点击预设"每天凌晨" → 自动填充表达式 `0 0 * * *`
4. 查看预览：中文描述 + 下次执行时间
5. 切换到手动模式修改表达式 → 图形化选择器同步更新

**预期结果**：图形化配置正常工作，预览准确。
