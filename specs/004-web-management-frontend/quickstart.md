# 快速入门验证指南：Web 管理前端界面

**功能**：004-web-management-frontend | **日期**：2026-06-19

## 前提条件

- Java 21 + Maven（或 Docker）
- AList-Media-Sync 后端服务已构建并运行（端口 8080）
- 至少一个可用的现代浏览器（Chrome / Firefox / Edge）

## 启动服务

```bash
# 方式一：Maven 直接启动
mvn spring-boot:run

# 方式二：Docker 启动
docker compose up -d
```

服务启动后，访问 `http://localhost:8080/app/` 进入 Web 管理界面。

## 验证场景

### 场景 1：登录认证

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 访问 `http://localhost:8080/app/` | 自动跳转到登录页面（`/app/#/login`） |
| 2 | 不输入任何内容，点击登录 | 提示"请输入用户名和密码" |
| 3 | 输入错误密码，点击登录 | 页面显示"用户名或密码错误"，用户名输入框不清空 |
| 4 | 输入正确凭据（默认 admin / admin123），点击登录 | 跳转到仪表板页面 |

### 场景 2：存储引擎管理

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 点击侧边栏"存储引擎" → 点击"添加引擎" | 弹出创建表单 |
| 2 | 输入引擎名称、AList API 地址、用户名、Token | — |
| 3 | 点击"保存" | 新引擎出现在列表中，密码列显示 `***` |
| 4 | 点击新引擎的"测试连接"按钮 | 5 秒内显示绿色"连接成功"或红色"连接失败" |
| 5 | 点击"编辑"修改引擎名称 → 保存 | 列表中名称更新 |
| 6 | 点击无引用引擎的"删除"→ 确认 | 引擎从列表中移除 |
| 7 | 点击已被任务引用的引擎的"删除" | 显示"该引擎正被使用，无法删除"及引用详情 |

### 场景 3：同步任务创建与执行

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 确保已有至少 2 个存储引擎 | 列表中有可选引擎 |
| 2 | 点击"同步任务" → "创建任务" | 弹出创建表单 |
| 3 | 填写任务信息，同步模式选择"仅新增"，执行计划选择"手动触发" | — |
| 4 | 点击"保存" | 新任务出现在列表中 |
| 5 | 点击任务的"立即执行"按钮 | 任务开始执行 |
| 6 | 在任务详情页观察进度 | 显示已完成/总文件数、当前文件、速度、预估时间 |
| 7 | 切换到"执行历史"标签 | 表格显示本次执行记录 |

### 场景 4：cron 表达式预览

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 编辑同步任务，执行计划选择"cron 表达式" | cron 输入框出现 |
| 2 | 输入 `0 */6 * * *` | 输入框旁显示"每 6 小时执行一次"，下次执行时间预览 |
| 3 | 输入 `invalid cron` | 输入框旁显示红色"cron 表达式格式错误" |

### 场景 5：转码任务管理

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 点击"转码任务" → "创建转码任务" | 弹出创建表单 |
| 2 | 选择源引擎、目标引擎，填写文件路径，格式选择 MP3 | — |
| 3 | 点击"保存" | 新转码任务出现在列表，状态为"等待中"或"转码中" |
| 4 | 对于上传失败的任务，点击"重试上传" | 状态更新 |
| 5 | 点击"清理残留文件"按钮 | 显示清理文件数量 |

### 场景 6：Webhook 规则和事件

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 点击"Webhook 规则" → "创建规则" | 弹出创建表单 |
| 2 | 填写规则名称，选择触发事件、后续动作 | — |
| 3 | 点击"保存" | 新规则出现在列表中 |
| 4 | 点击规则的"禁用"开关 | 规则标记为禁用 |
| 5 | 点击"Webhook 事件" | 显示事件列表或空状态提示（含配置引导） |

### 场景 7：仪表板

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 点击侧边栏"系统概览" | 显示统计卡片（活跃任务、等待任务、今日处理、成功率） |
| 2 | 点击"活跃同步任务"卡片 | 跳转到同步任务列表 |
| 3 | 当无活跃任务时 | 显示"系统当前空闲" |

## 边界情况验证

| 场景 | 操作 | 预期行为 |
|------|------|---------|
| 会话过期 | 登录后等待 30 分钟不操作 → 刷新页面 | 自动跳转到登录页，显示"会话已过期，请重新登录" |
| 空数据状态 | 删除所有引擎后访问引擎页面 | 显示"暂无存储引擎"及引导按钮"创建第一个引擎" |
| 网络中断 | 启动服务后手动停止后端 → 在页面上操作 | 显示"连接失败，正在重试"提示 |
| 页面刷新恢复 | 在任务详情页（`/app/#/sync-tasks/3`）按 F5 刷新 | 页面重新加载后仍显示同一任务详情 |
| 表单校验 | 在创建引擎表单中留空必填字段 → 提交 | 字段下方显示红色错误提示"此项为必填" |
| 删除确认 | 删除一个同步任务 | 弹出确认对话框，显示任务名称 |
